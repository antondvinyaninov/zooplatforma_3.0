# Changelog (Technical)
Все заметные технические и архитектурные изменения в проекте будут задокументированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

## [Unreleased]

## [3.1.3] - 2026-04-02
### Added
- **Frontend / Org:** Реализована логика и UI-компонент передачи прав владельца организации (`OrgStaffProfile.tsx`), интеграция с бэкенд обработчиком `/:id/transfer`.
- **Frontend / Pets:** Реализована валидация при переводе питомца в статус "В каталоге" — теперь обязательна загрузка хотя бы одной фотографии (`PetProfileLayout.tsx`).
- **Frontend / Pets:** Реализована логика автоматического назначения первой загруженной фотографии в галерее в качестве главной (`is_main`), если у питомца еще нет главного фото (`PetGallery.tsx`).
### Changed
- **Frontend / Org:** Расширено применение `short_name` вместо `name` во внутренних интерфейсах личного кабинета и панели управления (layout навигация, заголовки, списки).
- **Frontend / Org:** Замена стаб-данных (mock) в компоненте `AddStaffDrawer.tsx` на реальный API поиск пользователей с дебаунсом.
- **Frontend / Org:** В реестре организаций (`/orgs`) и карточках опекуна `AuthorCard` внедрено отображение `short_name` (с фоллбэком на `name`) для оптимизации UI пространства. Устранено склеивание ссылок логотипов (внедрен `getOrgLogoUrl`).
- **Backend / Pets:** SQL-запросы `GetByID` и `GetCatalog` переведены на использование конструкции `COALESCE(NULLIF(o.short_name, ''), o.name, u.name) as owner_name`.
- **Frontend / Pets:** Реорганизация макета карточки питомца `PetPageClient.tsx`. Блок описания `DescriptionCard` перенесен в левую/основную колонку под галерею и перед публикациями (Apple-Way).
### Fixed
- **Frontend / Catalog:** Исправлена проблема жесткой привязки фильтра регионов к строке города в `page.tsx`. Внедрена валидация вхождения города в список городов выбранного региона (`citiesData.regions`), что решает "проблему пропавших питомцев", когда API отдает только город без приписки региона.
- **Frontend / Catalog:** Исправлено отображение счетчика просмотров (`views_count`). Внедрен реальный проброс статистики из базы через обработчик `GetCatalog` вместо хардкода нуля, добавлена иконка просмотров в сетку "списком".
- **Frontend / Auth:** Изменен ошибочный путь авторизации `/api/gateway/auth/login` на единый шлюз бэкэнда `/api/auth/login` в кабинете зоопомощника (`pethelper/auth/page.tsx`). Восстановлен успешный редирект в `/pethelper/pets` и работа функции `checkAuth` через `/api/auth/me`.
- **Frontend / Auth:** Решена проблема несовпадения гидратации (Hydration Mismatch) и блокировок CORS от VK SDK на `localhost` путем создания безопасной заглушки `Mounted` для Dev-окружения (`VKIDButton.tsx`).
- **Frontend / Org:** Внедрена централизованная функция `getMediaUrl` для безопасного резолва смешанных S3 URL на странице `/orgs/[id]` и старой форме редактирования `/orgs/[id]/edit`.
- **Backend / Org:** Исправлена 500 ошибка (invalid input syntax for type date) при создании/обновлении организации с пустой датой регистрации.
- **Backend / Org:** Устранен хардкод `userID := 1` во всех обработчиках `organizations/handler.go` (внедрен хелпер динамического извлечения `getUserID` из контекста Gin).
- **Backend / Org:** Исправлена 500 ошибка Audit Log при передаче прав владельца организации (устранен мисматч колонок `actor_id` -> `admin_id` в SQL-запросе вставки в `admin_logs`).
- **Backend / Auth:** Исключен консольный "шум" (красные ошибки 401 Unauthorized) для гостевых проверок сессии. Эндпоинты `/me` и `/login` теперь возвращают `200 OK` с `success: false` в случае неавторизованного доступа.
- **Frontend / Org:** Исправлен баг в `/orgs/[id]/page.tsx` с некорректным отображением кнопок управления владельца (исправлена строгая проверка типов `string` vs `number` в `isMember`).

## [3.1.2] - 2026-03-31
### Added
- **Backend / Org:** Расширена модель организации и DTO `UpdateOrganizationData` (добавлено 9 новых полей ссылок), обновлен обработчик `GetByID` для возврата расширенной структуры `organizations-api.ts`.
- **Database:** Выполнена миграция базы данных: таблица `organizations` расширена новыми колонками для мессенджеров и соцсетей (vk, max, telegram_channel и др.).
- **Frontend / Org:** Выполнена интеграция новых 9 полей социальных сетей и мессенджеров (включая российский мессенджер Max и интеграцию Rutube) в интерфейс редактирования организации (`EditableRow`).
- **Backend / Org:** Реализованы полноценные CRUD эндпоинты управления сотрудниками (GET, POST, PUT, DELETE) в роутере `internal/org/router.go`.
- **Database:** Выполнена миграция БД: добавлены колонки `org_avatar` (тип TEXT) и `permissions` (тип JSONB) в таблицу `organization_members`.
- **Frontend / Org:** Выполнена интеграция компонента `OrgStaffList` с реальным Go-бэкендом.
- **Frontend / Org:** Компонент `AddStaffDrawer` подключен к бекенду для отправки реального инвайта в штат по email.
- **Frontend / Org:** Детальный профиль сотрудника (`OrgStaffProfile`) переведен с мокового localStorage на боевой бэкенд, включая загрузку корпоративного фото на S3 через `POST /api/media/upload` и сохранение прав доступа (modules).
- **Frontend / Profile:** Интегрирована зависимость `react-easy-crop` и реализован полнофункциональный модальный UI компонент `ImageCropperModal.tsx` для кадрирования изображений через Canvas API.
- **Frontend / Profile:** Внедрен Apple-Way Image Cropper в форму загрузки логотипа и обложки профиля организации с поддержкой автогенерации оптимизированных Blob-файлов 1:1 и 4:1.
- **Frontend / Public Org:** Полный редизайн компонента `Команда` на публичной странице организации (`/orgs/[id]`). Реализована горизонтальная скролл-карусель (Apple/Netflix style) с edge-to-edge карточками, градиентами для читаемости белого текста и fallback-механизмом аватарок.
- **Backend / Public Org:** Метод `GetMembers` (public API) в роутере `internal/mainfrontend/organizations` расширен для возврата поля `org_avatar`.
- **Frontend / API:** В логике формирования ссылок на изображения исправлен баг с двойным присоединением базового URL (S3) для внешних ссылок.
- **Frontend / Public Org:** Внедрен Apple-Way компонент навигации (Pill-shaped Tabs) на странице организации (`/orgs/[id]`) для переключения между "Инфо", "Команда" и "Подопечные" с теневым выделением активной вкладки.
- **Frontend / Public Org:** Рефакторинг модуля контактов/адресов. Контакты (Телефон, Telegram, Почта, Сайт) перенесены в отдельный блок правого сайдбара (Apple Quick Action Row - круглые значки). Локация выделена в секцию "Где мы находимся" внутри вкладки "Информация".
- **Frontend / Org Dashboard (`org/[id]/organization`):** Полностью переработан механизм ввода физического адреса. 
  - Для поля `address_city` внедрен компонент `CityAutocomplete` (DaData).
  - Для поля `address_full` внедрен полноэкранный интерактивный `LocationPicker`. При выборе адреса маркером происходит атомарное обновление на бэкенде гео-координат (`geo_lat`, `geo_lon`) и форматированной строки (`address_full`).
- **Frontend / Public Org:** Добавлена перманентная отрисовка ленты "Посты организации" снизу страницы, независимо от выбранной вкладки.
- **Backend / Org:** Добавлена логика поиска сотрудников `GET /api/org/:orgId/users/search` (поиск по name, email, id с исключением уже состоящих в организации).
### Fixed
- **Backend / S3:** Исправлена генерация публичных URL S3-объектов `FirstVDS` (переход с Virtual-Hosted на Path-Style формат устранил ошибку `ERR_NAME_NOT_RESOLVED`).
- **Frontend / Public Org:** Редизайн заголовка блока "Подопечные питомцы" приведен к единому строгому стандарту блока "Команда" (удален градиент, добавлены иконки Heroicons и унифицированные бейджи).
- **Frontend / Public Org:** Улучшен дизайн вкладок: заменены системные эмодзи на SVG-иконки `@heroicons`, добавлены видимые контрастные фоновые подложки (`bg-gray-100/80` для неактивных и `bg-white shadow` для активных).
## [3.1.1] - 2026-03-28
### Changed
- **Frontend / Cabinet:** Рефакторинг дашбордов Владельца и Зоопомощника. Убраны избыточные плашки со статистикой и независимая страница дашборда; внедрен прямой редирект с `/(dashboard)/dashboard` на `/(dashboard)/pets` для сокращения пользовательского пути.
- **Frontend / Layout:** Снят жесткий ограничитель ширины контейнеров `max-w-7xl` (`PetsListLayout`), теперь рабочие области и таблицы платформы (B2B/B2C) адаптивно применяют full-width Edge-to-Edge рендеринг на widescreen мониторах.

### Removed
- **Frontend / Cabinet:** Из `app/owner` и `app/pethelper` удалены неиспользуемые и блокирующие UI вызовы API `/organizations`, а также дублирующая вкладка "Главная" из бокового меню `layout`.

## [3.1.0] - 2026-03-27
### Added
- **Frontend / Feed:** Реализован интеллектуальный механизм инжектирования виджетов внутрь `virtualizer` бесконечной ленты постов (smart feed injections).
- **Frontend / Feed:** Добавлена врезка `PetCatalogCarousel` (карусель каталога) на индекс 2 для кросс-опыления с каталогом питомцев `looking_for_home`.
- **Frontend / Feed:** Добавлена нативная CTA-врезка `PromoActionCard` на индекс 6 для конвертации гостей во владельцев и волонтеров.
- **Frontend / Feed:** Разрешена "Проблема холодного старта" (Cold Start) — если для `activeFilter === 'city'` API возвращает пустой массив на нулевом offset, клиент принудительно запрашивает глобальную ленту без фильтра и выводит информационный баннер `isFallbackActivated`.
- **Frontend / Feed:** Внедрено сокращение длинных текстов постов (`line-clamp-5`) с кнопкой "Показать полностью..." для разгрузки UI ленты.
- **Frontend:** Добавлено отображение `org_pet_number` во внутренних и публичных карточках.
- **Frontend:** Добавлена новая ленивая вкладка "Инфо" для мобильной навигации (`PetNavMenu.tsx`).
- **Frontend:** Добавлена поддержка `as="textarea"` в компоненте `EditableRow`.
- **Architecture / Backlog:** Утверждена Глобальная Стратегия Роста (Virality + Tool-Led Growth). В `BACKLOG.md` интегрированы новые архитектурные модули (ЗооАкадемия, PetPassport, Сборы, NPS, User Telemetry и др.).
- **Frontend / Sharing:** Внедрена система нативного шэринга карточек через `Web Share API` (`navigator.share`) с graceful fallback'ом.
- **Backend / Next.js:** Развёрнут Edge API роут `app/api/og/route.tsx` с использованием `@vercel/og` для динамической генерации 1200x630 OpenGraph-превью постов (Vercel Satori).
- **Frontend / SEO:** Добавлена динамическая генерация `generateMetadata` в корневом `page.tsx` для автоматической сборки `<meta property="og:image">` при глубоком связывании (deeplinking) по параметру `?metka=`.
- **Backend:** В SQL-запрос `GET /:orgId/pets/:petId` (файл `org/router.go`) добавлены колонки `city` и `actual_city`.

### Changed
- **Frontend / Messenger:** Выполнен полный редизайн UI/UX мессенджера (Apple UI / Telegram style). Заменены плоские списки на плавающие карточки (floating chips) в `ChatList`. Поле ввода переработано в отделенный `Floating Dock` с эффектом матового стекла `backdrop-blur-xl`.
- **Frontend / Messenger:** Улучшен рендеринг бабблов сообщений в `MessageList`. Внедрены градиентные фоны, объемные тени `shadow` и усовершенствована типографика.
- **Frontend / Messenger:** Оптимизирован рендеринг медиа-вложений (Edge-to-Edge). При отправке standalone фото/видео синий контейнер баббла скрывается, медиа принимает радиус `rounded-[24px]` с абсолютным оверлеем прозрачного таймстемпа.
- **Frontend / Messenger:** Устранен визуальный баг "double-bubble" для файловых вложений (DOCX/PDF). Файлы рендерятся нативно внутри основного пузыря. Компонент прикрепленного питомца обновлен до премиального Rich-preview формата.
- **Frontend / UI:** Внедрена унифицированная дизайн-система `Post-Card` для всех карточек (посты, виджеты, профили, модальные окна). Произведен отказ от агрессивного glassmorphism и скруглений `rounded-2xl` в пользу строгого стандарта: `rounded-lg` (8px), `border-gray-200`, `shadow-sm`.
- **Frontend / Profile:** Стили внутренних карточек питомца (`PetNavMenu`, `PetGeneralInfo`, `PetGuardianCard`, `AddPetModal`) синхронизированы с единым стандартом UI.
- **Frontend / Home:** Редизайн Hero-блока и сайдбара (`HomeClient.tsx`, `FeedFilters.tsx`) приведен к единому Post-Card интерфейсу.
- **Frontend:** Полный рефакторинг `PetInfoCard` c переходом от вертикального списка к адаптивной двухколоночной сетке виджетов.
- **Frontend:** Реорганизация DOM-структуры `PetPageClient.tsx` (кондициональный рендеринг) для спуска ленты публикаций (`PublicationsList`) под информационные блоки на мобильных устройствах.
- **Frontend:** В `MobilePetProfileLayout` удален статический блок "Главное", инфо-поля инкапсулированы в динамическую вкладку.

### Removed
- **Frontend:** Исключена устаревшая дублирующаяся вкладка "Место содержания" из мобильного представления модулей.
- **Frontend:** Удалена заглушка `defaultDescription` из `DescriptionCard.tsx` (если поле пустое, блок скрывается).

### Fixed
- **Frontend:** Исправлен баг со сбросом локального стейта поля "Город" (очистка после обновления страницы).
- **Frontend:** Устранены проблемы с переполнением Tailwind-сеток (grids) в карточках информации и таблицах на узких экранах.
- **Frontend:** Восстановлено исчезнувшее поле редактирования "Описание питомца (для каталога)" во внутренней конфигурации профиля (`PetGeneralInfo.tsx`).
