# Workflow: Команда "запиши"

При вызове пользователем слова **"запиши"** (или аналогичной просьбы зафиксировать проделанную работу), вы как AI-агент должны выполнить следующие действия:

1. Ознакомиться со списком последних выполненных задач (например, в файле `task.md` или проанализировав текущий контекст диалога).
2. Вычленить из технического контекста:
   - Суть измененного/нового функционала для разработчиков (касается архитектуры, API, компонентов).
   - Суть изменений для конечных пользователей (визуальный улучшенный UX, новые возможности, багфиксы).
3. Добавить записи в файл `/CHANGELOG.md` в текущую секцию версии:
   - Использовать строгий технический формат (категории `Added`, `Changed`, `Removed`, `Fixed`).
   - Например: `- **Frontend / Cart:** Оптимизирован рендеринг корзины (memoization)`.
4. Добавить записи в файл `/RELEASES.md` в текущую секцию версии:
   - Использовать понятный, продуктовый язык, ориентированный на пользователя платформы. Сфокусироваться на пользе (ценности) изменения, а не на технической реализации (без `API`, `useEffect`, `React`).
   - Использовать подходящие эмодзи и группировать изменения по двум секциям: `### 🎉 Что нового` и `### ✨ Улучшения и исправления`.
   - Например: `* **Мгновенная корзина:** Мы полностью переписали алгоритм корзины, теперь товары добавляются мгновенно без зависаний! 🎉`
5. **ЖЕСТКОЕ ПРАВИЛО ДАТ И ВЕРСИЙ:** Если текущая дата (сегодняшний день) отличается от даты последнего релиза в логах, вы **НЕ ИМЕЕТЕ ПРАВА** дописывать в старую версию. Вы ОБЯЗАНЫ создать новую версию-заголовок (например, перевести `3.1.2` в `3.1.3`) с новой текущей датой, и записывать изменения только туда.
6. Сообщить пользователю, что изменения успешно зафиксированы в обоих логах.
