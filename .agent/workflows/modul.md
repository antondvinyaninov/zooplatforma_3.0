---
description: Начать разработку нового модуля или вернуться к прерванному модулю (Создает/переключает ветку feature/...)
---
При вызове этой команды пользователь передает название модуля (например, `/modul crm` или "Делаем Модуль CRM"). Вы должны выполнить следующие шаги:

// turbo-all
1. Если в проекте есть несохраненные изменения (измененные файлы), закоммитьте их в **текущую** ветку: `git add . && git commit -m "wip: save state before switching module"` (согласия пользователя не спрашивать, если коммит фейлится из-за отсутствия изменений — это нормально).
2. Выполните проверку, существует ли уже такая ветка: `git branch --list feature/[название]`
3. Если ветка **существует**, переключитесь на нее: `git checkout feature/[название]` (это означает возврат к прерванному модулю).
4. Если ветки **не существует**, перейдите в main, обновитесь и создайте её: `git checkout main && git pull origin main && git checkout -b feature/[название]`
5. Сообщите пользователю об успешной подготовке среды для работы над модулем.
